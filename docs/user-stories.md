# 用户故事 — 食分 v1.0

> 日期：2026-07-08 | 格式：User Story | 总计 15 条

---

## 核心功能

### US-01 — AI 配料解读
**作为** 关注食品健康的消费者，**我想要** 输入食品配料表后立刻获得通俗解读，**以便** 快速判断这东西能不能吃、安不安全。

**验收条件**：
- [x] 文本框输入配料表，点击「开始解读」调用 DeepSeek API
- [x] 返回 JSON 结构：一句话总结、健康评分(1-10)、成分逐条解读、总体建议、标签
- [x] API 未配置或调用失败时显示具体错误提示
- [x] 提供 3 个示例配料表供快速体验

### US-02 — 成分详情展开
**作为** 消费者，**我想要** 看到每种成分的详细信息和风险等级，**以便** 了解哪些成分需要特别注意。

**验收条件**：
- [x] 每条成分显示名称、分类、风险等级（绿/黄/红）、通俗描述
- [x] 标注适合/不适合人群（孕妇、儿童、糖尿病人群等）
- [x] 风险等级支持多种 AI 措辞（安全/注意/慎用、低/中/高等）

## 社区互动

### US-03 — 社区分享
**作为** 用户，**我想要** 把解读结果分享到社区，**以便** 帮助其他人避坑或发现好产品。

**验收条件**：
- [x] 分析结果页点击「分享到社区」创建公开帖子
- [x] 帖子包含头像、昵称、配料原文、评分、总结
- [x] 未登录用户点击分享时跳转登录页

### US-04 — 点赞
**作为** 用户，**我想要** 给社区里有帮助的帖子点赞，**以便** 表达认同。

**验收条件**：
- [x] 点击心形图标即时反馈（乐观更新）
- [x] 刷新后点赞状态和计数不丢失
- [x] 同一个人不能重复点赞

### US-05 — 评论
**作为** 用户，**我想要** 在帖子下评论讨论，**以便** 交流看法和提问。

**验收条件**：
- [x] 展开评论区查看所有评论
- [x] 登录后可发表评论（含昵称和头像）
- [x] 可删除自己的评论（确认弹窗）
- [x] 未登录用户提示登录

### US-06 — 编辑帖子
**作为** 发帖者，**我想要** 修改帖子的配料文字和总结，**以便** 纠正错误或补充信息。

**验收条件**：
- [x] 自己的帖子显示编辑按钮（社区 + 历史）
- [x] 弹出编辑框，可改配料文字和总结
- [x] 保存后社区和历史同步更新

### US-07 — 删除帖子
**作为** 发帖者，**我想要** 删除自己的帖子，**以便** 移除不想保留的内容。

**验收条件**：
- [x] 自己的帖子显示删除按钮（确认弹窗）
- [x] 删除后级联移除评论和点赞
- [x] 管理员可删除任何帖子

## 个人中心

### US-08 — 注册登录
**作为** 新用户，**我想要** 注册账号并登录，**以便** 使用社区功能。

**验收条件**：
- [x] 邮箱注册/登录（Supabase Auth）
- [x] 注册后自动登录
- [x] 封禁用户登录时显示提示并拒绝

### US-09 — 个人信息编辑
**作为** 用户，**我想要** 设置昵称和头像，**以便** 社区中展示个性化身份。

**验收条件**：
- [x] 12 个 emoji 可选作头像
- [x] 昵称最多 20 字
- [x] upsert 模式：有则更新，无则创建

### US-10 — 统计数据
**作为** 用户，**我想要** 看到我的分享数、获赞数、评论数，**以便** 了解自己的社区参与情况。

**验收条件**：
- [x] 个人页展示三个统计数字
- [x] 评论数可点击，跳转到「我的评论」
- [x] 数据与实际数据库一致

### US-11 — 解读历史
**作为** 用户，**我想要** 查看所有解读记录（含未分享的），**以便** 回溯之前的分析和已删除的内容。

**验收条件**：
- [x] 全部记录 Tab：本地未分享 + 已分享合并，按时间倒序
- [x] 未分享记录显示「分享到社区」按钮
- [x] 已分享记录可编辑和删除
- [x] 本地存储上限 30 条

### US-12 — 我的评论
**作为** 用户，**我想要** 查看我在别人帖子下的所有评论，**以便** 追踪我参与的讨论。

**验收条件**：
- [x] 「我的评论」Tab 显示所有评论
- [x] 每条评论显示内容、时间、头像、昵称
- [x] 「查看原帖」链接跳转到社区
- [x] 点击统计卡片「评论」直达此页面

## 后台管理

### US-13 — 仪表盘
**作为** 管理员，**我想要** 看到平台关键数据概览，**以便** 快速了解运营状况。

**验收条件**：
- [x] 显示总用户数、总帖子数、今日新增帖子、今日新增评论
- [x] 仅 admin 和 super_admin 可访问
- [x] 普通用户访问被重定向

### US-14 — 内容管理
**作为** 管理员，**我想要** 查看和删除全站帖子和评论，**以便** 维护社区内容质量。

**验收条件**：
- [x] 帖子列表：展开查看详情后删除任何帖子（确认弹窗）
- [x] 评论列表：查看原文、跳转原帖、删除任何评论
- [x] admin 和 super_admin 均可操作

### US-15 — 用户管理
**作为** 超级管理员，**我想要** 管理所有用户的角色、状态和信息，**以便** 控制平台权限和安全。

**验收条件**：
- [x] 用户列表显示昵称、头像、角色、状态
- [x] super_admin 可授/撤管理员、设置封禁、编辑用户信息
- [x] admin 只能管理普通用户的状态
- [x] 创建新用户（邮箱+密码）
- [x] 删除用户（super_admin 专属，不能删自己）
- [x] 不能修改 super_admin 的权限

### US-16 — OCR 拍照识别（v1.1）

**作为** 消费者在超市货架前，**我想要** 直接拍照配料表就能获得解读，**以便** 不用手动输入繁杂的配料文字。

**验收条件**：
- [x] 点击「拍照识别」调用手机相机
- [x] 照片送百度 OCR 识别文字
- [x] 识别结果自动填入输入框，可手动修正
- [x] OCR 凭证仅存服务端，前端不暴露

---

### US-17 — 条码扫描（v1.1）

**作为** 消费者在超市货架前，**我想要** 直接扫商品条码获取配料信息，**以便** 更快更准确地获得解读。

**验收条件**：
- [x] 点击「扫条码」启动摄像头
- [x] Chrome 原生 BarcodeDetector 自动识别条码
- [x] 查 Open Food Facts 获取配料
- [x] 配料自动填入输入框，可人工修正
- [x] 未收录商品提示拍照识别
