## 一、产品概述

### 1.1 产品定位
**「食分」是一款面向中国消费者的食品配料表解读工具。**

用户通过输入食品包装上的配料表文字，获得 AI 驱动的通俗化解读，包括成分安全性分析、适合人群判断、健康评分及饮食建议。

### 1.2 核心价值主张
- **解决什么问题**：消费者面对配料表「每个字都认识，看完不知道能不能吃」的信息不对称问题
- **为什么现在做**：健康消费意识觉醒（无糖、零添加、清洁标签趋势），但信息渠道杂乱，自媒体误导严重
- **差异化**：不做泛健康资讯，专注「配料表 → 能不能吃」这一精准场景

### 1.3 目标用户（按优先级）

| 优先级 | 用户群体 | 核心痛点 | 典型场景 |
|--------|----------|----------|----------|
| P0 | 宝妈 | 对孩子吃的东西高度敏感，信息搜集渠道杂乱、互相矛盾，容易焦虑 | 超市货架前拿起零食，翻看配料表，不确定添加剂对孩子的安全性 |
| P1 | 减脂/健身人群 | 需要控制热量和特定成分摄入，对「0糖」「低脂」标签存疑 | 选购代餐/运动饮料，想知道代糖方案是否健康、该喝多少 |
| P2 | 大众健康消费者 | 想吃得健康但懒得研究，需要快速、可信的一句话说清 | 日常采购时随手扫一眼，确认「这东西还行」即可放心购买 |

### 1.4 用户故事

| ID | 作为 | 我想要 | 以便 |
|----|------|--------|------|
| US-01 | 宝妈 | 输入孩子零食的配料表 | 知道里面的添加剂对孩子安不安全 |
| US-02 | 减脂用户 | 了解代糖的种类和风险 | 判断这瓶饮料是否该喝、喝多少 |
| US-03 | 所有用户 | 看到 1-10 分的健康评分 | 快速横向对比不同产品 |
| US-04 | 所有用户 | 看到成分逐条解读 | 了解每种成分是干什么的 |
| US-05 | 注册用户 | 分享解读结果到社区 | 帮助其他人避坑或发现好产品 |
| US-06 | 注册用户 | 浏览社区分享 | 看看别人发现了什么 |
| US-07 | 注册用户 | 点赞和评论 | 参与讨论、表达认同 |

## 二、功能规格

### 2.1 MVP 功能清单（v1.0）

#### F1 — 配料表输入
- **输入方式**：文本框手动粘贴 / 拍照 OCR 识别（v1.1）
- **拍照识别**：调用百度 OCR 提取配料表文字，自动填入输入框，支持手动修正
- **示例引导**：提供 3 个典型配料表示例（饼干、零糖饮料、奶粉）
- **校验**：非空校验，字数上限 2000 字

#### F2 — AI 解读（核心）
- **调用**：DeepSeek API，Prompt 模板化，要求返回 JSON
- **输出**：
  - 一句话总结（≤50 字）
  - 健康评分（1-10 分）+ 等级标签（健康/比较健康/一般/建议少吃）
  - 成分逐条解读（名称、分类、风险等级、通俗解释、适合/慎用人群）
  - 总体建议（30-100 字）
  - 标签（如「高蛋白」「低糖」「含代糖」）
- **容错**：API 异常时展示友好错误提示；解析失败时展示原始返回内容

#### F3 — 结果展示
- 评分卡片（颜色区分：绿/黄/红）
- 成分逐条卡片（左侧色条标识风险等级）
- 适合/慎用人群标签
- 总体建议高亮卡片
- AI 内容免责声明

#### F4 — 用户系统
- 邮箱注册/登录（Supabase Auth），登录状态持久化
- 未登录可浏览首页和社区，登录后才能分享和互动
- **个人信息编辑**：昵称 + emoji 头像选择，upsert 模式
- 新用户注册时触发器自动创建 profile

#### F5 — 轻量社区
- 帖子列表（按时间倒序，仅显示已分享帖子）
- 每条帖子展示：头像、昵称、配料原文、评分、一句话总结
- 展开查看成分详情（逐条成分 + 总体建议）
- 点赞（应用层 COUNT + UPDATE 维护计数，刷新恢复状态）
- 评论：展开查看、发表、删除自己的评论（确认弹窗）
- 删除帖子：作者可见，确认后删除（CASCADE 评论和点赞）
- 编辑帖子：作者可改配料文字和总结，两边（社区+历史）一致

#### F6 — 个人中心
- 统计数据（分享数、获赞数、评论数）——评论数可点击跳转
- 「最近分享」预览（3 条，查看全部 → 历史记录）
- **历史记录**页面：全部记录 + 我的评论（Tab 切换）
- 全部记录：本地未分享 + 已分享合并显示，按时间倒序
- 未分享记录可「分享到社区」，已分享可编辑和删除
- 我的评论：显示自己的所有评论 +「查看原帖」链接
- 本地存储上限 30 条（localStorage）

### 2.2 v1.1 计划功能

- **F8 — 社区图文化**：帖子附带实物包装照片

### 2.3 v1.2 计划功能

### 2.4 v2.0 远期规划

## 三、非功能需求

### 3.1 性能
- 首页加载 ≤ 2s（Lighthouse 评分 ≥ 80）
- AI 解读响应 ≤ 10s（展示 loading 动画）
- 首屏 JS bundle ≤ 150KB（gzip）

### 3.2 可靠性
- AI API 调用失败时展示降级提示，不白屏
- Supabase 未配置时各页面展示引导提示而非崩溃
- 所有组件覆盖加载中、空数据、错误三种非正常状态

### 3.3 安全
- API Key 仅存于服务端环境变量，前端不暴露
- 用户密码由 Supabase 托管加密
- 社区内容不含用户敏感信息

### 3.4 合规
- AI 生成内容标注「仅供参考，不构成医学建议」
- 食品添加剂安全性评级基于公开国家标准（GB 2760）

## 四、成功指标

| 指标 | 目标（MVP 阶段） | 衡量方式 |
|------|-------------------|----------|
| 核心功能可用 | 输入配料表 → 获得解读，全流程可用 | 手动测试 |
| AI 解读质量 | 解读结果与配料表实际成分对得上 | 抽样对比 10 个真实配料表 |
| 基础留存 | 社区有 > 3 条真实分享 | 数据库查询 |
| 项目完整度 | 产品文档 + 代码 + 可演示 Demo | 简历可展示 |
