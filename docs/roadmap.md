## v1.1 — OCR 拍照 + 图片社区

**目标**：拍照识别配料表 + 社区图文化，提升活跃度

| 功能 | 优先级 | 说明 |
|------|--------|------|
| OCR 拍照识别配料表 | ✅ 已完成 | 百度 OCR → Vercel 函数代理 → 自动填入输入框 |
| 手动编辑识别结果 | ✅ 已完成 | OCR 结果填入后可手动修正再解读 |
| 图片分享到社区 | ⏳ 待做 | 帖子可附带实物包装照片 |
| 图片存储（Supabase Storage） | ⏳ 待做 | 图片上传与存储 |

## v1.2 — 后台管理系统

**目标**：内容管理 + 数据看板

| 功能 | 说明 |
|------|------|
| 内容审核面板 | 图文内容审核（图片分享上线后的刚需） |
| 用户管理 | 违规处理、用户状态管理 |
| 数据看板 | 用户增长、活跃度、解读次数等关键指标 |

## v2.0 — 家常菜 AI 视觉识别

**目标**：从「配料表解读」升级为「食物视觉识别平台」

| 功能 | 说明 |
|------|------|
| 拍照识别家常菜 | 拍成品菜 → AI 识别食材、估算用量、分析营养 |
| 多模态 AI | 计算机视觉 + 食材识别 + 营养估算，差异化壁垒极高 |
| 食谱反推 | 根据菜品照片推测做法和配料 |

**技术挑战**：需要视觉多模态模型支持，成本高于纯文本识别。建议 v1.1 先验证 OCR 场景，积累用户后 v2.0 再上。
|------|--------|------|
| OCR 拍照识别 | P0 | 调用 OCR API（如百度OCR / 腾讯OCR）识别配料表 |
| 手动编辑识别结果 | P0 | OCR 可能不准，允许修正后再解读 |
| 图片裁剪/旋转 | P1 | 提高识别准确率 |
| 解读历史记录 | P1 | 个人中心查看历史，按时间排列 |
| 评论功能 | P2 | 社区帖子下的评论 |

**技术选型考虑**：
- 百度 OCR（通用文字识别）：免费额度 1000 次/月
- 腾讯 OCR：免费额度 1000 次/月
- 如果用户量不大，免费额度够用

## v2.0 — 完整版（远期）

**目标**：从工具升级为平台

| 功能 | 说明 |
|------|------|
| 微信小程序版 | 降低使用门槛，方便传播 |
| 食品对比 | 并排对比两款产品的配料和评分 |
| 个性化偏好 | 设置过敏原、饮食偏好（素食/清真/低碳等），定制解读角度 |
| 食品添加剂百科 | 内置知识库，不依赖 AI 也能查 |
| 订阅提醒 | 关注某类产品/成分，新发现时推送 |
| 开放 API | 其他应用可接入解读能力 |

## 可能的商业化方向

| 方向 | 说明 |
|------|------|
| 品牌合作 | 清洁标签品牌付费展示「推荐替代」 |

## 个人成长路径（研发者视角）

这个项目的路线图也是你产品能力的成长路径：

1. **v1.0** → 简历项目：「从0到1独立设计开发产品，涵盖需求分析、用户画像、功能规划、数据库设计、前后端实现」
2. **v1.1** → 面试亮点：「基于用户反馈优化产品体验，引入 OCR 降低使用门槛，数据驱动迭代」
3. **v2.0** → 真正的产品经理能力：「平台化思考、商业化探索、多端适配」

面试官想看到的不是「我会写代码」，而是「我能发现问题 → 设计方案 → 落地执行 → 验证效果」这条链。
